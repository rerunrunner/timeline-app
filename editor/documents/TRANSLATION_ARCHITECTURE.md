# Multi-Language Translation Architecture

## Overview

This document outlines the architecture and implementation plan for adding multi-language translation support to the Timeline Data Editor. The system will support automatic AI-powered translation of reveal fields (displayed date, title, and description) into multiple languages.

## Database Schema

### New Tables

#### LANGUAGE Table
Stores supported languages for translation.

```sql
CREATE TABLE "PUBLIC"."LANGUAGE"(
    "ID" BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    "CODE" CHARACTER VARYING(10) NOT NULL,        -- 'en', 'es', 'ja', 'ko'
    "NAME" CHARACTER VARYING(100) NOT NULL,       -- 'English', 'Español'
    "IS_DEFAULT" BOOLEAN DEFAULT FALSE,
    "IS_ENABLED" BOOLEAN DEFAULT TRUE
);

ALTER TABLE "PUBLIC"."LANGUAGE" 
    ADD CONSTRAINT "PUBLIC"."CONSTRAINT_LANGUAGE_PK" PRIMARY KEY("ID");

CREATE UNIQUE INDEX "IDX_LANGUAGE_CODE" 
    ON "PUBLIC"."LANGUAGE"("CODE");

-- Initial data
INSERT INTO "PUBLIC"."LANGUAGE" ("CODE", "NAME", "IS_DEFAULT", "IS_ENABLED") 
VALUES 
    ('en', 'English', TRUE, TRUE),
    ('es', 'Español', FALSE, TRUE),
    ('ja', '日本語', FALSE, TRUE),
    ('ko', '한국어', FALSE, TRUE);
```

#### REVEAL_TRANSLATION Table
Stores translations for each reveal in different languages.

```sql
CREATE TABLE "PUBLIC"."REVEAL_TRANSLATION"(
    "ID" BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    "REVEAL_ID" BIGINT NOT NULL,
    "LANGUAGE_ID" BIGINT NOT NULL,
    "DISPLAYED_DATE" CHARACTER VARYING,
    "DISPLAYED_TITLE" CHARACTER VARYING,
    "DISPLAYED_DESCRIPTION" CHARACTER VARYING,
    "SOURCE" CHARACTER VARYING(20),               -- 'ai-auto', 'ai-manual', 'manual-edit', 'imported'
    "AI_MODEL" CHARACTER VARYING(50),             -- e.g. 'gpt-4o-mini', 'opus-mt-en-es'
    "CREATED_AT" TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    "UPDATED_AT" TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    "LAST_AI_TRANSLATED" TIMESTAMP,
    "LAST_MANUALLY_EDITED" TIMESTAMP,
    "AI_COST_ESTIMATE" DECIMAL(10,4)              -- Track approximate cost
);

ALTER TABLE "PUBLIC"."REVEAL_TRANSLATION" 
    ADD CONSTRAINT "PUBLIC"."CONSTRAINT_REVEAL_TRANSLATION_PK" PRIMARY KEY("ID");

ALTER TABLE "PUBLIC"."REVEAL_TRANSLATION" 
    ADD CONSTRAINT "FK_REVEAL_TRANSLATION_REVEAL" 
    FOREIGN KEY("REVEAL_ID") REFERENCES "PUBLIC"."REVEAL"("ID") ON DELETE CASCADE;

ALTER TABLE "PUBLIC"."REVEAL_TRANSLATION" 
    ADD CONSTRAINT "FK_REVEAL_TRANSLATION_LANGUAGE" 
    FOREIGN KEY("LANGUAGE_ID") REFERENCES "PUBLIC"."LANGUAGE"("ID");

CREATE UNIQUE INDEX "IDX_REVEAL_LANG" 
    ON "PUBLIC"."REVEAL_TRANSLATION"("REVEAL_ID", "LANGUAGE_ID");
```

#### TRANSLATION_USAGE Table
Track daily usage to prevent runaway costs.

```sql
CREATE TABLE "PUBLIC"."TRANSLATION_USAGE"(
    "DATE" DATE NOT NULL,
    "COUNT" INTEGER DEFAULT 0,
    "ESTIMATED_COST" DECIMAL(10,2) DEFAULT 0.00,
    "PROVIDER" CHARACTER VARYING(50)
);

ALTER TABLE "PUBLIC"."TRANSLATION_USAGE" 
    ADD CONSTRAINT "PUBLIC"."CONSTRAINT_TRANSLATION_USAGE_PK" PRIMARY KEY("DATE");
```

## Backend Services

### Service Layer Architecture

```
TranslationService (Interface)
├── translate(text, sourceLang, targetLang) → String
├── translateBatch(texts[], sourceLang, targetLang) → Map<String, String>
└── translateWithContext(text, context, sourceLang, targetLang) → String

Implementations:
├── HuggingFaceTranslationService ⭐ Recommended (Free)
│   ├── Uses Helsinki-NLP OPUS models
│   ├── Free API token from huggingface.co
│   ├── 1000 requests/hour free tier
│   └── Good quality neural translation
│
├── OpenAITranslationService (Best Quality)
│   ├── Uses gpt-4o-mini or gpt-4
│   ├── Context-aware narrative translation
│   ├── ~$0.15 per 1M tokens
│   └── Excellent for narrative or domain-specific content
│
└── LibreTranslateService (Fallback)
    ├── No API key required
    ├── Lower quality
    └── May be slow/unreliable
```

### Translation Manager Service

Orchestrates translation operations:

```java
@Service
public class TranslationManagerService {
    
    /**
     * Auto-translate reveal when saved (if enabled)
     * Only translates to languages where translation doesn't exist
     * or where source is 'ai' (manual edits preserved)
     */
    @Async
    public CompletableFuture<Void> autoTranslateReveal(Reveal reveal);
    
    /**
     * Translate reveal to specific language
     * Can be triggered manually from UI
     */
    public RevealTranslation translateRevealToLanguage(
        Reveal reveal, 
        String targetLanguageCode
    );
    
    /**
     * Bulk translate multiple reveals
     * Background job for translating entire dataset
     */
    @Async
    public CompletableFuture<BulkTranslationResult> bulkTranslateReveals(
        List<Reveal> reveals, 
        String targetLanguageCode
    );
    
    /**
     * Re-translate all reveals (admin operation)
     */
    @Async
    public CompletableFuture<Void> retranslateAll();
}
```

### Auto-Translate Logic Flow

```
User saves Reveal (English fields changed)
    ↓
Is auto-translate enabled?
    ↓ YES
For each enabled language (ES, JA, KO, etc.):
    ↓
    Does translation exist?
        ↓ NO → Create new AI translation
        ↓ YES → Check translation source
            ↓ source = 'ai-auto' or 'ai-manual'
            │   → Regenerate (English changed)
            ↓ source = 'manual-edit'
            │   → SKIP (preserve user edits)
            ↓ source = 'imported'
                → Check if older than reveal.updated_at
                    → If older, flag for review (UI warning)
    ↓
Save translations to REVEAL_TRANSLATION table
    ↓
Return success
```

## Export Strategy

### Hybrid File Approach (Recommended)

Export one base file plus language overlay files:

```
export/
├── dataset.json              # Base file with English (default)
├── dataset-es.json           # Spanish translations only
├── dataset-ja.json           # Japanese translations only
└── dataset-ko.json           # Korean translations only
```

### Base File Structure (`dataset.json`)

Contains full data structure with default language (English):

```json
{
  "version": "20240315-123456",
  "languages": ["en", "es", "ja", "ko"],
  "defaultLanguage": "en",
  "metadata": {
    "id": "my-timeline",
    "name": "My Timeline",
    "version": "20240315-123456"
  },
  "events": [
    {
      "id": "TL1-event1",
      "timelineId": "TL1",
      "shortDescription": "Quarterly review meeting",
      "reveals": [
        {
          "id": "1",
          "episodeId": "2",
          "episodeTime": 930,
          "absolutePlayTime": 930,
          "screenshotFilename": "reveal-001.jpg",
          "displayedDate": "March 15, 2024",
          "displayedTitle": "Quarterly review meeting",
          "displayedDescription": "The team completes the milestone. Follow-up is scheduled for next week."
        }
      ]
    }
  ]
}
```

### Language Overlay File Structure (`dataset-es.json`)

Contains only translations, much smaller file:

```json
{
  "language": "es",
  "version": "20240315-123456",
  "translations": {
    "reveal-1": {
      "displayedDate": "15 de marzo de 2024",
      "displayedTitle": "Reunión de revisión trimestral",
      "displayedDescription": "El equipo completa el hito. La reunión de seguimiento está programada para la próxima semana."
    },
    "reveal-2": {
      "displayedDate": "5 de enero de 2024",
      "displayedTitle": "Otro evento",
      "displayedDescription": "..."
    }
  }
}
```

### File Size Estimates

For ~N reveals with 3 fields each (e.g. 170):

```
dataset.json       ~500 KB   (full structure + English)
dataset-es.json    ~50 KB    (translations only)
dataset-ja.json    ~60 KB    (CJK characters slightly larger)
dataset-ko.json    ~60 KB    (CJK characters slightly larger)
```

### Viewer Integration

The timeline viewer loads data as follows:

```javascript
// 1. Load base dataset
const baseData = await fetch('dataset.json').then(r => r.json());

// 2. If user selects non-default language, load overlay
if (selectedLanguage !== baseData.defaultLanguage) {
  const overlay = await fetch(`dataset-${selectedLanguage}.json`)
    .then(r => r.json());
  
  // 3. Apply translations by reveal ID
  baseData.events.forEach(event => {
    event.reveals.forEach(reveal => {
      const translation = overlay.translations[`reveal-${reveal.id}`];
      if (translation) {
        reveal.displayedDate = translation.displayedDate;
        reveal.displayedTitle = translation.displayedTitle;
        reveal.displayedDescription = translation.displayedDescription;
      }
      // If translation missing, falls back to default (English)
    });
  });
}

// Now baseData contains data in selected language
```

## Translation Providers

### Provider Comparison

| Provider | API Key? | Quality | Speed | Cost | Korean Support |
|----------|----------|---------|-------|------|----------------|
| **Hugging Face** ⭐ | Free token | ⭐⭐⭐⭐ | Fast | Free | Very Good |
| **OpenAI** | Paid | ⭐⭐⭐⭐⭐ | Very Fast | ~$2-3 total | Excellent |
| **LibreTranslate** | No | ⭐⭐⭐ | Slow | Free | Fair |

### Recommended: Hugging Face (Free)

**Why Hugging Face:**
- ✅ Free API access with generous limits (1000 req/hour)
- ✅ Good quality neural machine translation
- ✅ Reliable service (99.9% uptime)
- ✅ Easy setup (30 second signup)
- ✅ Supports all major languages
- ✅ Production-ready

**Setup Steps:**
1. Go to [huggingface.co](https://huggingface.co)
2. Sign up (email + password)
3. Navigate to Settings → Access Tokens
4. Create new token (read access)
5. Copy token (format: `hf_xxxxxxxxxxxxx`)
6. Paste into app settings

**Models to Use:**
- English → Spanish: `Helsinki-NLP/opus-mt-en-es`
- English → Japanese: `Helsinki-NLP/opus-mt-en-ja`
- English → Korean: `Helsinki-NLP/opus-mt-en-ko`
- English → French: `Helsinki-NLP/opus-mt-en-fr`

**API Example:**
```bash
POST https://api-inference.huggingface.co/models/Helsinki-NLP/opus-mt-en-es
Authorization: Bearer hf_xxxxxxxxxxxxx
Content-Type: application/json

{
  "inputs": "The team completes the milestone"
}

Response:
{
  "translation_text": "El equipo completa el hito"
}
```

### Optional: OpenAI (Best Quality)

**When to Use:**
- Need context-aware translation
- Working with narrative or story content
- Domain-specific or specialized terminology
- Budget allows (~$2-3 for typical dataset)

**Cost for a Typical Dataset:**
- e.g. ~170 reveals × 3 fields × 4 languages = 2,040 translations
- Average ~30 tokens per translation
- ~60,000 tokens total ≈ **$1-2** using gpt-4o-mini

## Cost Analysis

### One-Time Translation Cost

For translating an entire dataset once:

**Example dataset:**
- e.g. 170 reveals
- 3 fields per reveal (date, title, description)
- 4 target languages (ES, JA, KO, FR)
- **Total: e.g. 2,040 translation calls**

**Provider Costs:**

| Provider | Cost Model | One-Time Cost | Ongoing |
|----------|------------|---------------|---------|
| **Hugging Face** | Free (1000/hr) | **$0** | $0 |
| **OpenAI (gpt-4o-mini)** | $0.15/1M tokens | **$1-2** | ~$0.01 per update |
| **OpenAI (gpt-4)** | $3/1M tokens | **$10-15** | ~$0.20 per update |
| **LibreTranslate** | Free | **$0** | $0 |

### Ongoing Costs

**Scenario: Auto-translate on save**
- Assume 10 reveals edited per day
- 3 fields × 4 languages = 12 translations per reveal
- 120 translations per day

| Provider | Daily Cost | Monthly Cost | Annual Cost |
|----------|------------|--------------|-------------|
| **Hugging Face** | $0 | $0 | $0 |
| **OpenAI (gpt-4o-mini)** | ~$0.02 | ~$0.60 | ~$7 |
| **LibreTranslate** | $0 | $0 | $0 |

### Recommendation

**Start with Hugging Face:**
- Zero cost
- Good quality
- Production-ready
- Easy to upgrade to OpenAI later if needed

**Upgrade to OpenAI if:**
- You need strong context-aware translation
- Working with complex narrative content
- Budget allows (e.g. ~$10/year for active editing)

## Security Considerations

### API Key Storage

**Backend Storage:**
- Store encrypted in database
- Use AES-256 encryption
- Never expose raw key to frontend
- Backend makes all API calls

```java
// Example encryption approach
@Column(name = "translation_api_key")
@Convert(converter = EncryptedStringConverter.class)
private String translationApiKey;
```

### Rate Limiting

**Prevent Abuse:**
- Daily translation limit (default: 1000)
- Monthly translation limit (default: 10,000)
- Track usage in TRANSLATION_USAGE table
- Alert when approaching limits

### Cost Control

**Safeguards:**
- Required API key for paid services
- Usage tracking and reporting
- Configurable limits
- Bulk operation confirmations
- Manual approval for large batches
